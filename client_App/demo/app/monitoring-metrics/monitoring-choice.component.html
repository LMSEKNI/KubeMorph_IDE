Step-by-Step Guide

1. Clone the Metrics-Server Repository

First, you need to clone the official metrics-server repository to get the necessary deployment files.

git clone https://github.com/kubernetes-sigs/metrics-server.git
 cd metrics-server

2. Deploy the Metrics-Server

Kubernetes provides a default deployment configuration that you can use directly.

kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

3. Verify Installation

Ensure that the metrics-server pods are running correctly.

kubectl get pods -n kube-system

Look for metrics-server in the list. You should see one or more metrics-server pods running.

4. Modify Deployment (Optional)

If you're running a custom Kubernetes cluster or facing issues like TLS certificate errors, you might need to modify the deployment configuration. Edit the components.yaml to add the following arguments under the containers.args section of the metrics-server deployment:

- --kubelet-insecure-tls
- --kubelet-preferred-address-types=InternalIP,Hostname,InternalDNS,ExternalDNS,ExternalIP

Apply the modified configuration:

kubectl apply -f components.yaml

If you are Using Minikube

Deploy Metrics Server

By default, the metrics server is not installed in the Minikube environment. You can deploy the metrics server by running the following command:

minikube addons enable metrics-server
